# 备件管理系统 UI 设计规范

## 1. 目标与适用范围

本规范用于统一备件管理系统的页面布局、主题色、卡片、表格、筛选区、表单和图片附件上传体验。新增页面或调整现有页面时，应优先复用全局主题、公共 CSS 类和已有组件，不在单页重复定义同类视觉规则。

设计基线来自“申购记录”页面的优点：清晰分组、柔和背景、轻量卡片、低干扰表格和明确的主操作。改进后的原则是减少页头说明文字，只保留对当前操作真正有帮助的信息。

## 2. 设计原则

1. **信息优先**：视觉层级服务于业务信息，不使用无意义的装饰和重复说明。
2. **页头克制**：列表页页头只显示标题和必要操作；不在标题下重复解释页面名称。
3. **一致分组**：筛选、数据、详情、编辑等区域使用统一卡片容器。
4. **单一主色**：交互主色统一使用蓝色，绿色只表达成功状态。
5. **复用优先**：相同交互必须复用公共组件或全局样式，不复制页面级实现。

## 3. 设计令牌

设计令牌同时维护在 `frontend/src/theme.ts` 和 `frontend/src/styles.css`。

### 3.1 颜色

| 用途 | 变量 | 色值 |
| --- | --- | --- |
| 主色 | `--color-primary` | `#3f63d8` |
| 主色悬停 | `--color-primary-hover` | `#5376e4` |
| 主色浅背景 | `--color-primary-soft` | `#edf2ff` |
| 成功 | `--color-success` | `#229b6b` |
| 警告 | `--color-warning` | `#d99020` |
| 危险 | `--color-danger` | `#d94b64` |
| 强文本 | `--color-text-strong` | `#172033` |
| 正文 | `--color-text` | `#4b5565` |
| 弱文本 | `--color-text-muted` | `#7d8798` |
| 页面背景 | `--color-bg` | `#f4f6fa` |
| 卡片背景 | `--color-surface` | `#ffffff` |
| 弱化背景 | `--color-surface-soft` | `#f8faff` |
| 边框 | `--color-border` | `#e2e7ef` |
| 弱边框 | `--color-border-subtle` | `#edf0f5` |

禁止用绿色表示普通主操作；绿色仅用于完成、启用、正常等成功语义。

### 3.2 圆角与阴影

- 输入框、选择器和按钮使用 `9px` 左右圆角。
- 卡片使用 `14px` 圆角。
- 普通卡片使用轻阴影 `--shadow-card`，避免高对比度或大面积深阴影。
- 仅允许可交互的统计卡片在悬停时使用 `--shadow-card-hover`。

### 3.3 间距

- 页面内容区：上/左右/下分别为 `24px / 28px / 32px`。
- 页面一级区块间距：`18px`。
- 卡片内部默认间距由 Naive UI 主题统一控制，通常为 `20px 22px`。
- 表单栅格列间距：`20px`；卡片栅格间距：`18px`。

## 4. 页面结构

### 4.1 页头

统一使用：

```html
<div class="page-header">
  <div>
    <h1 class="page-title">页面标题</h1>
  </div>
  <n-space>页面操作</n-space>
</div>
```

- 列表页默认不使用 `page-subtitle`。
- 详情页可在标题下显示一行关键元数据，例如编号、状态、时间或型号。
- 元数据最多一行，不写操作说明、技术实现或业务规则。
- 业务规则应放入对应卡片、表单提示或 `n-alert` 中。

### 4.2 卡片

- 页面内容统一使用 `n-card`。
- 筛选卡片增加 `filter-card`，使用轻微蓝灰渐变。
- 数据表卡片增加 `data-card`，统一表格上方和分页间距。
- 不在单页重复设置卡片圆角、阴影和边框。

### 4.3 筛选区

- 简单筛选使用 `filter-bar`，控件间距为 `10px`。
- 多条件筛选可使用“申购记录”的栅格形式，并保留一个明确的“筛选条件”标题。
- 不显示“多个条件将组合查询”等可由用户自然理解的说明。
- 文本搜索支持多关键词时，统一使用半角 `|` 表示“或”，并兼容全角 `｜`；不要在每个输入框的占位文字中重复提示该规则。
- 同一输入框内的多个关键词按 OR 匹配，不同筛选控件之间仍按 AND 组合。
- 主按钮使用“查询”或“查询记录”，重置为次按钮并与查询按钮相邻。

### 4.4 数据表格

- 表头使用浅灰蓝背景和中等字重。
- 行悬停使用淡蓝背景，不改变文字颜色。
- 分页放在卡片底部右侧，并使用顶部分隔线。
- 行点击可进入详情时，操作列仍应保留明确的文字操作或保证鼠标状态可感知。
- 列宽按内容类型分级，统一引用 `frontend/src/constants/table.ts`：
  - 单位、状态、数量等短字段使用约 `80–110px`。
  - 日期、人员、编码等中等字段使用约 `112–176px`。
  - 名称使用约 `220px`，型号规格使用约 `240px`。
  - 同时展示名称和型号的物资摘要使用约 `280px`。
  - 用途、原因等长文本至少使用 `200px`，并提供省略提示。
- 每个数据表列同时设置 `width` 与相同的 `minWidth`，从组件层面禁止浏览器继续压缩。
- `scroll-x` 必须按当前实际列宽计算；当全部列宽超过内容区域时启用横向滚动，不得通过等比例压缩牺牲名称和型号列的可读性。
- 使用 `n-table` 的普通表格必须放入 `table-scroll` 容器，并设置符合字段数量的最小宽度。

### 4.5 详情与编辑页

- 详情和编辑内容按业务分组为多个卡片。
- 卡片标题使用业务名词，如“单据信息”“物资明细”，避免“基本信息 1”之类名称。
- 两列内容使用 `form-grid`；主信息加侧栏使用 `detail-grid`。
- 详情页返回入口统一放在内容上方的 `detail-toolbar` 左侧，使用次要按钮并明确返回目标，不使用孤立的文字链接。
- 当卡片标题已经能说明页面内容时，不再额外重复物资名称、编号、日期、状态等大块页头摘要。
- 编辑表单只保留一个“保存修改”按钮，默认放在表单卡片底部右侧；顶部工具栏仅放返回、删除、状态流转等页面级操作。
- 关联物资、选择单位等表单字段统一随整表保存，不为单个字段增加独立保存或确认流程。
- 破坏性操作不得与主要保存按钮使用同一种视觉权重。

### 4.6 顶栏账号信息

- 右上角账号入口使用“用户名 + 角色”的两级文字结构，用户名为主信息，角色使用弱文本。
- 系统没有用户头像能力时，不显示首字母头像或占位头像图标，避免制造不存在的个性化能力预期。
- 账号入口默认保持轻量，仅在悬停和键盘聚焦时使用主色浅背景与弱边框。
- 下拉入口必须保留清晰的可交互状态和焦点样式，不使用高对比度实心按钮抢占页面主操作层级。

### 4.7 二级库信息分工

- “库存查询”是二级库菜单的第一入口，集中展示当前库存、低库存状态和库存操作。
- “物资档案”只展示名称、型号规格、计量单位等档案信息，不在列表重复展示库存数量。
- 物资档案不设计启用、停用状态及相关筛选、按钮或状态标签，操作列只保留编辑入口。
- 筛选项必须与当前用户权限一致；无权限访问的数据状态应直接从选项中移除，并由后端同步校验。

## 5. 图片附件上传

图片附件统一使用 `frontend/src/components/ImageUploader.vue`，不得直接在页面内新增原生文件选择器或另一套上传卡片。

统一要求：

- 表单标签统一写作“图片附件”。
- 支持 JPG、PNG、WebP，单张不超过 10 MB，默认最多 9 张。
- 支持点击选择和 `Ctrl+V` 粘贴。
- 缩略图、删除按钮、禁用状态、上传中状态由组件统一处理。
- 页面只负责通过 `v-model:files` 绑定文件列表，不自行拼接预览地址或删除逻辑。

示例：

```html
<n-form-item label="图片附件">
  <ImageUploader v-model:files="images" />
</n-form-item>
```

## 6. 状态与反馈

- 主操作：主题蓝色。
- 成功、启用、已完成：绿色。
- 风险提醒、低库存：黄色或橙色。
- 删除、失败、不可逆操作：红色。
- 普通统计标签优先使用中性色或主题浅色，不滥用状态色。
- 加载、成功、失败反馈优先使用 Naive UI 的 message、notification、dialog 和 loading 状态。

## 7. 实现约束

1. Naive UI 主题覆盖集中在 `frontend/src/theme.ts`。
2. 跨页面布局和视觉类集中在 `frontend/src/styles.css`。
3. 表格列宽令牌集中在 `frontend/src/constants/table.ts`。
4. 单页 `<style scoped>` 只保留该页面独有的布局或交互样式。
5. 颜色必须优先使用 CSS 变量或主题令牌，避免新增硬编码色值。
6. 新增重复 UI 前先检查 `frontend/src/components` 是否已有可复用组件。

- [ ] 卡片是否复用全局圆角、边框和阴影。
- [ ] 筛选卡片是否使用 `filter-card`。
- [ ] 表格卡片是否使用 `data-card`。
- [ ] 状态色是否符合语义，而非用于普通装饰。
- [ ] 图片附件是否复用 `ImageUploader`。
- [ ] 是否通过 `npm run build`、`npm run test` 和 `npm run lint`。
