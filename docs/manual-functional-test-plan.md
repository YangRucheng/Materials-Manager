# 电气车间备件管理系统人工功能测试方案

## 1. 测试目标与范围

验证当前版本各业务功能、权限控制、数据校验及库存—申购—到货闭环是否符合设计。

- 测试方式：人工黑盒测试，仅通过浏览器页面操作和页面结果判断。
- 测试范围：登录与权限、工作台、二级库物资、库存、出入库及流水、申购计划、未编码物资、申购记录、到货入库、计量单位、用户管理、图片和 Excel 导出。
- 不在范围：性能、压力、安全渗透、兼容性专项、接口自动化、单元测试及部署运维测试。

## 2. 测试准备

### 2.1 环境与账号

- 使用已初始化且前后端服务正常的独立测试环境。
- 浏览器允许下载文件，测试人员可访问系统上传图片。
- 初始账号及密码：`admin`、`warehouse`、`purchase`、`readonly`，密码均为 `123456`。

| 角色 | 预期权限 |
| --- | --- |
| 超级管理员 | 全部读写权限、计量单位和用户管理 |
| 仓库管理员 | 二级库物资、库存策略、补库、入库、出库、流水维护；申购模块只读 |
| 申购管理员 | 申购计划及申购记录维护；库存模块只读 |
| 只读角色 | 全部业务数据只读，无新增、编辑、删除、出入库入口 |

### 2.2 基础数据

准备以下数据，名称建议增加本轮测试批次前缀，避免与历史数据混淆：

- 计量单位：一个整数单位（小数位 0）、一个允许 1 位小数的单位。
- 二级库物资：正常库存、零库存、低库存物资各一项。
- 图片：合法 JPG/PNG/WebP；非图片文件；大于 10 MB 的图片；至少 10 张小图片。
- 申购计划：有编码、无编码、已关联库存物资、未关联库存物资各一条。
- 已转申购记录：未到货、部分到货、全部到货各一条。

## 3. 执行规则

- 优先级：P0 为主流程或核心数据正确性，P1 为重要分支，P2 为一般查询或提示。
- 每条用例记录实际结果、通过/失败、测试人和日期；失败时记录账号、数据、步骤、页面提示及截图。
- 除用例明确要求外，新增/修改成功后均刷新页面并重新查询，确认数据已持久化。
- 库存允许为负数；历史流水修改后会重算其后流水及库存，不应将负库存本身判为缺陷。
- 申购到货允许超过计划数量；累计到货达到或超过计划数量后状态均应为“已完成”。

## 5. 核心回归链路

每次修复或版本发布至少执行以下 P0 链路：

1. 仓库管理员新建二级库物资，设置库存预警并完成初始入库。
2. 出库至低库存，确认工作台和低库存列表出现预警，发起补库计划。
3. 申购管理员补全物料编码，导出采购申请表并转为申购记录。
4. 仓库管理员分两次到货，确认库存、流水、部分到货和完成状态正确。
5. 修改第一次到货流水数量，再执行冲销，确认库存及申购进度均按流水重算。
6. 分别用四种角色复查菜单、按钮和直接地址访问权限。

## 6. 通过准则

- 所有 P0 用例通过，且不存在阻断主流程或造成库存/申购数据错误的缺陷。
- P1 用例全部执行；遗留问题已评估且不影响上线范围。
- 核心回归链路数据前后一致，刷新或重新登录后结果不丢失。
